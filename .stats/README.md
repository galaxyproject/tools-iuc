# IUC Toolshed Deployment Statistics

Generated by [`stats/main.py`](../stats/main.py) which parses all `<tool id="...">` from `tools/`, resolves version macros at each git commit, and classifies each change along the `origin/main` first-parent chain (4520 commits, 2014-09-13 to 2026-06-29).

## Output Files

| File | Description |
|---|---|
| `tool_events.csv` | 8381 events (tool_id, version, change_type, commit_hash, commit_date, etc.) |
| `tool_events.json` | Same data in JSON |
| `summary.txt` | Year-by-year breakdown of new tools and updates |
| `cumulative_tools.png` | Cumulative tools over time (logistic-like S-curve) |
| `combined_monthly.png` | Monthly new + update bars on shared axis |
| `new_tools_per_month.png` | New tool additions per month |
| `updates_per_month.png` | Version updates per month |
| `activity_heatmap.png` | Year × month heatmap of all activity |
| `top_updated_tools.png` | Top-20 tools by update count |
| `change_type_breakdown.png` | Upstream vs wrapper vs version changes |

## Punchlines

- **1660 unique tools** tracked (of 1664 total in repo; 4 have no deployment history on main)
- **8381 total deployment events**: 1660 new tool additions, 6721 version updates
- **Breakdown of updates**: 2797 upstream version bumps, 3806 wrapper revisions, 118 version-only changes (tools using custom tokens like `@STACKS_VERSION@` or hardcoded versions, not classifiable through the standard token system)
- **~2 new tools added and ~11 updates deployed per week** on average (all time)
- Activity has **doubled in the last year**: 25.1 events/week (last year) vs 13.6/week (all time)
- **3.6 events/day** in the last year
- **Most updated tool**: `datasets_download_genome` (35 updates); 7 of top 10 are HyPhy tools
- Activity peaks in **2024** (1020 updates), with a secondary peak in **2020** (839 updates — pandemic-era bioinformatics surge)

## Running the analyzer

The analyzer lives in `.stats/scripts/` and is packaged with `pyproject.toml` so it works with `uv`.

### With uv (recommended)

```bash
cd .stats
uv sync
uv run tools-iuc-stats --repo .. --verbose
```

### With pip in a virtual environment

```bash
cd .stats
python -m venv .venv
source .venv/bin/activate
pip install -e .
tools-iuc-stats --repo .. --verbose
```

### Useful options

```bash
# Skip plot generation for a faster run
tools-iuc-stats --repo .. --verbose --skip-plots

# Run analysis in parallel (4 workers)
tools-iuc-stats --repo .. --verbose --workers 4

# Write outputs to a custom directory
tools-iuc-stats --repo .. --output-dir ./output
```

## Graph ideas

Possible additions or improvements:

1. **12-month rolling average** on the monthly activity bars to highlight long-term trends.
2. **Cumulative updates over time** — currently only cumulative *new tools* is shown; a second S-curve for total updates would show maintenance velocity.
3. **Monthly update-to-new ratio** — a line plot of updates ÷ new tools, showing when the project shifted from growth to maintenance.
4. **Update type timeline** — stacked area chart of `upstream`, `wrapper`, and `version` changes per quarter.
5. **Top categories** — bar chart of new tools and updates grouped by `.shed.yml` `categories`.
6. **Tool age distribution** — histogram of years between a tool's first appearance and its most recent update.
7. **Top contributor suites** — bar chart of events grouped by top-level tool directory (e.g. `hyphy`, `bedtools`, `qiime`).
8. **Wrapper-churn leaderboard** — tools with the most `wrapper`-only updates, highlighting high-maintenance wrappers.
9. **Activity by day of week / seasonality** — reveals team cadence and hackathon surges.
10. **Interactive HTML report** — replace static PNGs with a single `altair`/`plotly` page that supports zooming and filtering.
